③ 3 COMPONENTES
O que você ganha

Três componentes, zero repetição

Um único dicionário de campos alimenta tudo

📋
<dyf-form>
Formulário CRUD
Cria e edita registros. Campos com validação, máscara, select e agrupamento por abas.
📊
<dyf-grid>
Grid de listagem
Tabela com colunas automáticas, paginação, busca e ações de editar/excluir/detalhe.
🔍
<dyf-detail>
Detalhe readonly
Exibe um registro formatado para leitura, com labels e valores dos campos configurados.
④ COMO FUNCIONA
Fluxo

Três passos para ter um CRUD

Do JSON ao produto final em minutos

1

Defina o dicionário

Crie um objeto DyfTable com os campos, tipos e opções. Use o Builder ou escreva na mão.

2

Use os componentes

Passe o DyfTable para <dyf-form>, <dyf-grid> ou <dyf-detail> via @Input.

3

CRUD pronto

DynaForm gera formulários, colunas e layouts automaticamente. Nada de código repetitivo.

⑤ QUICK START (código de exemplo)
Exemplo

Começar leva 5 minutos

Instale, defina o dicionário, use os componentes

TypeScript — dicionário
const TABLE: DyfTable = {
  tableName: 'clientes',
  label: 'Clientes',
  fields: [
    {
      fieldName: 'id',
      label: 'ID',
      type: 'number',
      key: true,
    },
    {
      fieldName: 'nome',
      label: 'Nome',
      type: 'string',
      required: true,
      gridColumns: 8,
    },
    {
      fieldName: 'ativo',
      label: 'Ativo',
      type: 'boolean',
      gridColumns: 4,
    },
  ],
};
HTML — template
<!-- Grid de listagem -->
<dyf-grid
  [table]="TABLE"
  [items]="items"
  (edit)="onEdit($event)"
  (delete)="onDelete($event)">
</dyf-grid>

<!-- Formulário de edição -->
<dyf-form
  [table]="TABLE"
  [value]="item"
  (save)="onSave($event)"
  (cancel)="back()">
</dyf-form>

<!-- Detalhe readonly -->
<dyf-detail
  [table]="TABLE"
  [value]="item">
</dyf-detail>
